# Fire Fit Gym 利用ガイド

Fire Fit Gym の顧客・セッション・売上を管理するアプリです。
このドキュメントはアプリを使う人（オーナー・スタッフ）向けの操作ガイドです。

## 1. 初期セットアップ

### 1-1. アプリにログインする

1. アプリのURL（例: `https://ffg-client-base-xxxxxxxx.vercel.app`）を開く
2. ログイン画面が出るので **新規登録** タブを選ぶ
3. メールアドレスとパスワード（6文字以上）を入力し **登録** をクリック
4. 画面に「以下のSQLを実行してください」というメッセージが出るので、表示されたSQL文をコピーして手元にメモ

> このアプリは現在 Google ログインも対応していますが、設定が必要なため、当面はメール+パスワードで使います。

### 1-2. Supabase でメール確認をスキップする

通常、新規登録するとメール確認が必要ですが、テスト中はSQLで一発で承認できます。

1. ブラウザで [Supabase](https://supabase.com) を開いてログイン
2. 該当のプロジェクトを開く
3. 左サイドバーの 📝 アイコン（**SQL Editor**）をクリック
4. **New query** をクリックして、以下を貼り付け（メアド部分を自分のに置き換え）：

```sql
update auth.users
set email_confirmed_at = now()
where email = 'あなたのメール@example.com';
```

5. 画面右下の **Run** をクリック → `Success. No rows returned` または `1 row` のような表示が出ればOK

### 1-3. 自分をオーナー権限にする（最初の1回だけ）

ジムのオーナーは「売上ダッシュボード」と「プラン管理」を使うために**オーナー権限**が必要です。
スタッフのみなら不要です。

同じSQL Editorで：

```sql
update public.profiles
set role = 'owner'
where email = 'あなたのメール@example.com';
```

→ **Run** をクリック。

### 1-4. アプリに戻ってログイン

1. アプリのログイン画面に戻る
2. **ログイン** タブを選ぶ
3. さきほど登録したメアドとパスワードを入力し **ログイン**
4. カレンダー画面が表示されたら成功

画面上部に5つのメニューが出ているはずです：
- カレンダー
- 当日確定
- 顧客
- 売上
- プラン

> 売上・プランが見えていない人は、1-3 のオーナー権限SQLが実行されていない可能性があります。

### 1-5. プランを確認・調整

サンプルとして以下4つのプランが入っています：

| プラン名 | 回数 | 価格 |
|---|---|---|
| 16回券（通常） | 16回 | ¥80,000 |
| 16回券（継続割引） | 16回 | ¥72,000 |
| 8回券 | 8回 | ¥44,000 |
| ペア16回券 | 16回 | ¥120,000 |

実際の価格に合わせて編集してください：

1. 上部メニュー **プラン** をクリック
2. 編集したいプランをタップ
3. 価格や回数を編集して **保存**

不要なプランは「販売中」のチェックを外せば、チケット販売時の選択肢から消えます。

## 2. 具体的な操作方法

### 2-1. 顧客を登録する

1. 上部メニュー **顧客** → 右上 **新規** ボタン
2. 入力項目：
   - **名前**: 必須
   - **タグ**: カンマ区切りで「紹介, 継続」など任意
   - **メモ**: 体調、目標、紹介者などフリーテキスト
3. **登録する** をクリック

### 2-2. 顧客にチケットを販売する

1. 顧客一覧から該当顧客をタップ → 詳細画面
2. 右上 **チケット販売** ボタン
3. 入力：
   - **対象**: 個人顧客（ペア組のチケットは後述）
   - **顧客**: 自動で選ばれている
   - **プラン**: 購入したプランを選択
   - **購入日**: 今日（デフォルト）
   - **回数**: プラン既定でOK（空欄）
   - **個別単価**: 通常価格と違う場合のみ入力（例: 継続割引で¥72,000）
   - **メモ**: 任意
4. **登録する**

→ 顧客詳細画面の「残チケット」が増えていれば成功。

### 2-3. セッションの予約を登録する

1. 上部メニュー **カレンダー**
2. **空いている時間帯をタップ**（PCはドラッグ、スマホは長押しで時間を選択）
3. ダイアログが開くので：
   - **開始 / 終了**: 自動で入っているので必要なら調整
   - **参加者**: 顧客を選択 → そのお客様の使えるチケットを選択
   - **メモ**: 任意
4. **保存**

→ カレンダー上に青いブロックで予定が表示されます。

#### 予定の移動

予定をドラッグすれば時間変更できます。

#### 予定の編集・削除

予定をタップ → ダイアログが開く → 編集または **削除** ボタン。

### 2-4. ペアセッション

ペアセッションは2パターンあります。

#### パターンA: ペア専用プラン（1枚で2人分）

**事前準備（1回だけ）**：

1. 上部メニュー **顧客** → URLを直接編集して `/pair-groups` にアクセス
   （または `/customers` のURL末尾を `/pair-groups` に変更）
2. **新規登録** で：
   - **組名**: 「田中ご夫妻」など
   - **顧客A / 顧客B**: それぞれ選択
3. **登録する**

**チケット販売**：

1. 上部メニュー **顧客** → どちらかの顧客の詳細画面 → **チケット販売**
2. **対象** を「ペア組」に切り替え
3. **顧客 / ペア組** で「田中ご夫妻」を選択
4. プランは「ペア16回券」を選択
5. **登録する**

**セッション予約**：

1. カレンダーで時間帯をタップ
2. 参加者1: 顧客Aを選択 → 使うチケットで「ペア16回券（残○○）」を選択
3. **+ ペアでもう1名追加** をクリック
4. 参加者2: 顧客Bを選択 → 同じペアチケットを選択
5. 「ペアプランの共有チケットとして1枚消費されます」と表示されたらOK
6. **保存**

#### パターンB: 通常チケット×2枚を同時消費（例外組）

両者ともに通常プランを購入している場合：

1. カレンダーで時間帯をタップ
2. 参加者1: Aさん → Aさんの通常チケット
3. 参加者2: Bさん → Bさんの通常チケット（**別の**チケット）
4. **保存**

→ 確定時に2枚消費されます。

### 2-5. 当日確定（1日の終わりの作業）

セッションを行ったら、消化扱いにします。

1. 上部メニュー **当日確定**
2. 当日のセッションが一覧で出る
3. 各セッションに3つのボタン：

| ボタン | 何が起こる |
|---|---|
| **実施** | 通常通りセッションが行われた → チケット1枚消化、売上計上 |
| **当日キャンセル(消化)** | お客様都合の当日キャンセル → チケット消化、売上計上（実施と同じ扱い） |
| **キャンセル(返却)** | やむを得ない事情で振替 → チケット減らず、売上0 |

> 体調不良などはケースバイケースで「実施」or「返却」を選んでください。判断はオーナーの裁量です。

#### 確定を間違えた場合

確定済みのセッションには「確定を取り消す」ボタンが出ます。
これを押すと、消化したチケットが戻り、売上もリセットされ、未確定状態に戻ります。

#### 過去の未確定が溜まったとき

`/today` 画面上部に「未確定の過去セッション」が表示されます。
押し忘れがあれば、ここから遡って確定してください。

### 2-6. 顧客の残チケットを確認

- 上部メニュー **顧客** → 各顧客の名前の横に残枚数が出る
- 顧客の詳細を開くと、チケット履歴・セッション履歴も見られる

### 2-7. 月別売上を見る（オーナーのみ）

- 上部メニュー **売上**
- 過去12ヶ月の月別売上が棒グラフで表示される
- 「セッション消化時」に計上される稼働ベース売上です

## 3. テストで確認する内容

新しい人（友人）にテストしてもらう場合、以下のシナリオを順に試して、すべてうまく動けばOKです。

各テストには「期待される結果」を書いてあります。違う場合は記録してください。

### ✅ チェックリスト

#### A. 認証

- [ ] **A-1**: 新規登録 → SQLで承認 → ログインできる
- [ ] **A-2**: ログイン後、画面上部にメニューが出る
- [ ] **A-3**: ログアウトすると `/login` に戻る
- [ ] **A-4**: ログアウト状態で `/calendar` を直接URLに入れても `/login` に飛ばされる

#### B. プラン

- [ ] **B-1**: 上部メニュー **プラン** に4つのプランが表示される
- [ ] **B-2**: プランをタップ → 編集 → 保存 → 変更が反映される
- [ ] **B-3**: 「販売中」のチェックを外すと、チケット販売画面に出てこなくなる

#### C. 顧客

- [ ] **C-1**: 新規顧客を1人登録（名前のみ）→ 一覧に出る
- [ ] **C-2**: 顧客詳細を開く → 残チケットが「0枚」と表示される
- [ ] **C-3**: 顧客を編集 → メモを追加 → 保存 → 詳細画面で反映される
- [ ] **C-4**: ステータスを「休会」に変更 → 一覧に「休会」バッジが出る

#### D. チケット販売

- [ ] **D-1**: 顧客詳細 → チケット販売 → 16回券を登録
- [ ] **D-2**: 顧客詳細の残チケットが **16枚** になっている
- [ ] **D-3**: 顧客一覧の右側にも **16枚** が出る

#### E. セッション予約

- [ ] **E-1**: カレンダーで空き時間をタップ → 顧客とチケットを選んで保存
- [ ] **E-2**: カレンダー上に予定が **青** で表示される
- [ ] **E-3**: 予定をタップ → 編集ダイアログが出る → メモを追加 → 保存できる
- [ ] **E-4**: 予定をドラッグして別の時間に移動できる
- [ ] **E-5**: 予定をタップ → 削除 → カレンダーから消える

#### F. 当日確定

事前準備: 今日の予定を3つ作る（顧客は同じでもOK）。

- [ ] **F-1**: **当日確定** メニューに3件並ぶ
- [ ] **F-2**: 1件目を **実施** → 「確定済み」になり、緑バッジに変わる
- [ ] **F-3**: 2件目を **当日キャンセル(消化)** → 同じく「確定済み」になる
- [ ] **F-4**: 3件目を **キャンセル(返却)** → グレーになる
- [ ] **F-5**: 顧客詳細を開く → 残チケットが **16 → 14枚** になっている（2枚消化）
- [ ] **F-6**: 当日の売上が画面上部に「¥10,000」など（実施したぶん）と出る
- [ ] **F-7**: 確定を取り消すと、残チケットが **14 → 15枚** に戻る

#### G. 月別売上（オーナーのみ）

- [ ] **G-1**: 上部メニュー **売上** をクリック → 今月の売上が表示される
- [ ] **G-2**: 過去12ヶ月の棒グラフが出る（実施したぶん）

#### H. ペアセッション（時間があれば）

事前準備: 2人の顧客を登録、ペア組として登録、ペア16回券を購入。

- [ ] **H-1**: カレンダーで予定作成 → 参加者1にAさん、ペアチケットを選択
- [ ] **H-2**: 「+ ペアでもう1名追加」 → 参加者2にBさん、**同じ**ペアチケットを選択
- [ ] **H-3**: 「ペアプランの共有チケットとして1枚消費されます」と表示される
- [ ] **H-4**: 保存 → カレンダーに表示される（タイトルに2人の名前）
- [ ] **H-5**: 当日確定で **実施** → ペアチケットが **16 → 15枚** に減る（1枚だけ）
- [ ] **H-6**: AさんとBさんの「セッション履歴」両方に表示される

#### I. レスポンシブ（スマホ・タブレット）

- [ ] **I-1**: スマホで開く → 画面下にメニューが出る
- [ ] **I-2**: カレンダーがスマホでも見やすい
- [ ] **I-3**: 顧客一覧・当日確定がスマホで操作できる

### 不具合があったとき記録してほしいこと

| 項目 | 例 |
|---|---|
| **どの画面で起きたか** | カレンダー / 顧客詳細 / 当日確定 / etc. |
| **何をしたか** | 「ペアチケットを選んでもう1名追加した」など具体的に |
| **何が起きるはずだったか** | 「ペアチケット1枚消費」 |
| **実際は何が起きたか** | 「ペアチケット2枚消費されてしまった」 |
| **デバイス** | iPhone / Android / PC / タブレット |
| **エラーメッセージ** | あればスクショ |

## 4. 困ったときは

### よくある症状と対処

| 症状 | 対処 |
|---|---|
| ログイン画面が出ない / 真っ白 | ブラウザのキャッシュをクリア / シークレットモードで試す |
| 「Email not confirmed」エラー | 1-2のSQLが未実行 |
| 「売上」「プラン」メニューが見えない | 1-3のオーナー権限SQLが未実行 |
| カレンダーで予定がタップしにくい | スマホは長押しで作成、タップで編集 |
| ペアチケットがない | 1. ペア組を登録、2. ペア組宛にチケット販売、の2ステップが必要 |
| Internal Server Error | デプロイ時の環境変数設定漏れの可能性。オーナーに連絡 |

### サポート連絡先

実装・運用に関する問い合わせは [リポジトリ管理者] までご連絡ください。

## 付録: 1日の典型的な使い方

**朝**
1. アプリを開いて **カレンダー** で今日の予定を確認

**セッション後 or 営業終了時**
1. **当日確定** メニューを開く
2. 今日のセッションを1件ずつ「実施 / 消化キャンセル / 返却キャンセル」で確定

**月末（オーナー）**
1. **売上** メニューで今月の数字を確認

**新規顧客が来店した時**
1. **顧客** → **新規** で登録
2. その顧客の詳細 → **チケット販売** で回数券を購入記録

**セッション中に次回の予約を取る時**
1. **カレンダー** で空き時間をタップ
2. 顧客を選んで保存
3. メモには「次回テーマ」など書いても良い
